A browser-native benchmark that compares AI judgments against real human court decisions — entirely on your device. Documents never leave the browser. Powered by WebGPU + WebLLM, with a heuristic baseline that runs anywhere.

Run models one at a time — each result is saved (in-browser, IndexedDB) and added to the leaderboard, which persists across reloads. Pick a model, run it, then pick the next: the leaderboard builds up your model comparison run by run. Loading a new LLM frees the previous one from GPU memory automatically.

Does the compression work? — judgment fidelity test

The core test of JudgeSaab: take real historical court cases, compress the facts, and re-run the AI judge. If the model returns the same judgment on the compressed facts as on the full facts, the compression preserved the legally-decisive information — it works. This is measured as judgment fidelity (verdict unchanged vs full context), independent of whether the AI agrees with the human judge. Runs every strategy (including Headroom, an adaptive query-aware compressor) on the selected model + dataset.

How it works

  1. Load case from a bundled dataset (ECtHR, SCOTUS, CaseHOLD…).
  2. Hide the human judgment; the model sees only the facts + question.
  3. Generate an AI judgment (verdict, reasoning, laws, confidence).
  4. Reveal the actual judgment and score across accuracy, reasoning, citation, hallucination, calibration, and consistency.
  5. Update the leaderboard.

Programmatic API on window.JudgeSaab: loadDataset(), runBenchmark(), runCase(), evaluate(), compareModels(), exportResults(). Bundled cases are original synthetic scenarios; real corpora can be added via the plugin API.
